# Сайт рецептов — Лабораторная работа по Django (Вариант 6)

Реализация платформы для публикации кулинарных рецептов на Python + Django с использованием PostgreSQL.

---

## Описание

Цель проекта — разработать веб-сайт, позволяющий пользователям просматривать и публиковать рецепты блюд.  
Сайт включает базовую аутентификацию и поиск по ингредиентам (дополнительное задание).

---

##  Страницы

- **Главная страница** — список рецептов с отображением:
  - Названия блюда  
  - Изображения  
  - Времени приготовления (в минутах)

- **Страница рецепта** — подробное описание:
  - Полный список ингредиентов с единицами измерения и количеством  
  - Пошаговая инструкция приготовления  
  - Возможность перехода с главной страницы по клику на название

- **Страница «О проекте»** — краткое описание целей и структуры сайта

- **Регистрация и вход** — стандартная система аутентификации

- **Поиск по ингредиентам** — позволяет найти все рецепты, содержащие указанный ингредиент

---

## ER-диаграмма базы данных (текстовое представление)

Согласно заданию, ER-диаграмма включает следующие сущности:

**Рецепт (Recipe):**
- recipe_id (PK)
- title
- cooking_time
- description
- img

**Ингредиенты (Ingredient):**
- ingredient_id (PK)
- name

**Множество_ингредиентов (RecipeIngredient):**
- id (PK)
- recipe_id (FK → Recipe)
- ingredient_id (FK → Ingredient)
- unit
- quantity

Отношения:
- Один рецепт → много записей в `RecipeIngredient`
- Один ингредиент → может использоваться во многих рецептах
- Связь **многие-ко-многим** между `Recipe` и `Ingredient` через промежуточную модель `RecipeIngredient`

- **Backend**: Python 3.11 + Django 5.2
- **База данных**: PostgreSQL
- **Frontend**: HTML5 + CSS3
- **Зависимости**: Pillow, python-decouple (для .env)
- **Версионирование**: Git + GitHub

## Установка
1. **Клонирование репозитория**n`git clone https://github.com/Aliya-aaa/trpo_laba_1.git
cd trpo_laba_1`
2. **Создание виртуального окружения:**

Создать`python -m venv venv`

Активировать `venv\Scripts\activate`
3. **Установка зависимостей**
`pip install -r requirements.txt`
4. **Настройка базы данных:**

Убедитесь, что PostgreSQL запущен.

Создайте базу данных recipes_db (например, через pgAdmin).

Обновите настройки подключения в файле .env (или в settings.py), указав:

Имя пользователя (DB_USER)

Пароль (DB_PASSWORD)

Пример .env:

DB_NAME=recipes_db

DB_USER=postgres

DB_PASSWORD=ваш_пароль

DB_HOST=localhost

DB_PORT=5432
5. **Применение миграций:** `python manage.py migrate`
6. **(Опционально) Создание суперпользователя:** `python manage.py createsuperuser`
7. **Запуск сервера:** `python manage.py runserver`
8. **Открытие в браузере:**

Перейдите по адресу:
http://127.0.0.1:8000
